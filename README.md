# NIDS — Network Intrusion Detection System

A machine-learning powered security dashboard that detects and classifies network intrusions in real time. Built on the **NSL-KDD** and **CICIDS2017** datasets using **XGBoost**, with a **FastAPI** backend, **PostgreSQL** storage, and a **React 19** frontend.

---

## Screenshots

> **Dashboard** — Stat cards, traffic overview, distribution donut, recent alerts, live simulation panel  
> **Alerts** — Sortable, filterable, paginated table with CSV export  
> **Predict** — Full 40-feature KDD form (or 69-feature CICIDS2017 form), confidence bar, session history  
> **Globe** — 3D interactive attack origin map  
> **Reports** — Weekly bar chart, monthly trend, category breakdown, PDF export  
> **Model Info** — Confusion matrix heatmap + feature importance chart  

---

## Models Supported

| Model | Dataset | Classes | Features | Accuracy |
|---|---|---|---|---|
| **NSL-KDD** | KDD Cup 1999 (revised) | 5 (Normal, DoS, Probe, R2L, U2R) | 40 | 80.22% test |
| **CICIDS2017** | Canadian Institute for Cybersecurity | 10 (Benign, DoS, DDoS, PortScan, BruteForce, Bot, Infiltration, Web Attacks) | 69 | 99.86% test |

Both models are XGBoost classifiers trained with SMOTE class balancing and tracked via MLflow.

## Setup Instructions

### Prerequisites

- Python 3.10+
- Node.js 18+
- PostgreSQL 14+ running locally

---

### 1 — Clone the repository

```bash
git clone https://github.com/Bhagyesh312/NIDS.git
cd NIDS
```

### 2 — Python virtual environment + dependencies

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate

pip install -r requirements.txt
```

### 3 — Configure the database

Create a PostgreSQL database named `NIDS` in pgAdmin4 or via psql:

```sql
CREATE DATABASE "NIDS";
```

Then set up the backend environment file:

```bash
cd backend
copy .env.example .env      # Windows
# cp .env.example .env      # macOS / Linux
```

Edit `backend/.env` with your credentials:

```env
DATABASE_URL=postgresql://postgres:YOUR_PASSWORD@localhost:5432/NIDS
```

Tables are auto-created on first startup — no migration script needed.

### 4 — Add raw datasets

Place the raw data files in `data/raw/` before running notebooks:

```
data/raw/KDDTrain+.txt
data/raw/KDDTest+.txt
data/raw/CICIDS2017/Friday-WorkingHours-Afternoon-DDos.pcap_ISCX.csv
data/raw/CICIDS2017/Friday-WorkingHours-Afternoon-PortScan.pcap_ISCX.csv
data/raw/CICIDS2017/Friday-WorkingHours-Morning.pcap_ISCX.csv
data/raw/CICIDS2017/Monday-WorkingHours.pcap_ISCX.csv
data/raw/CICIDS2017/Thursday-WorkingHours-Afternoon-Infilteration.pcap_ISCX.csv
data/raw/CICIDS2017/Thursday-WorkingHours-Morning-WebAttacks.pcap_ISCX.csv
data/raw/CICIDS2017/Tuesday-WorkingHours.pcap_ISCX.csv
data/raw/CICIDS2017/Wednesday-workingHours.pcap_ISCX.csv
```

**NSL-KDD**: [Canadian Institute for Cybersecurity](https://www.unb.ca/cic/datasets/nsl.html)  
**CICIDS2017**: [Canadian Institute for Cybersecurity](https://www.unb.ca/cic/datasets/ids-2017.html)

### 5 — Run the ML notebooks (in order)

```bash
jupyter lab

# Run notebooks in this order:
# 1. notebooks/01_eda_nslkdd.ipynb        (optional — EDA only)
# 2. notebooks/02_preprocessing.ipynb     (required — generates data/processed/)
# 3. notebooks/03_model_training.ipynb    (required — generates models/)
```

After notebook 03 completes you will have:
- `models/model.pkl` — trained NSL-KDD XGBoost classifier
- `models/model_info.json` — real accuracy metrics served by the API
- `models/cicids_model.pkl` — trained CICIDS2017 classifier (if CICIDS pipeline ran)

### 6 — Start the backend

```bash
cd backend
uvicorn main:app --reload
```

- API root: http://localhost:8000  
- Interactive docs: http://localhost:8000/docs  
- Redoc: http://localhost:8000/redoc  

### 7 — Start the frontend

```bash
cd frontend
npm install
npm run dev
```

Dashboard: http://localhost:5173

## Switching Between Models

The active model is controlled by the **NSL-KDD ↔ CICIDS2017** toggle in the header (available on every page) or via **Settings → ML Model**.

- All API calls automatically pass `?model=kdd` or `?model=cicids`
- The Predict page shows the correct feature form for each model
- Alerts, Stats, Globe, and Reports all filter by the active model
- The model selection is persisted to `localStorage`

## How the Notebook Pipeline Powers the Dashboard

```
notebooks/02_preprocessing.ipynb
    └── data/processed/*.npy + *.pkl
            ↓
    notebooks/03_model_training.ipynb
            ├── models/model.pkl            → ml.py loads at FastAPI startup
            │       └── POST /predict        → Predict page (single flow)
            │       └── POST /predict/batch  → Batch page (CSV upload)
            │       └── POST /simulate/start → Live simulation panel (Dashboard)
            │       └── DB predictions       → Alerts, Reports, Globe pages
            │
            └── models/model_info.json      → cached by stats.py on first read
                    └── GET /model-info      → ModelInfo page (matrix, importances)
                    └── GET /stats           → Dashboard Model Accuracy stat card
                    └── GET /reports/summary → Reports Detection Rate stat card
                    └── WelcomeModal         → val accuracy, feature count display
```

## Frontend Features

- **Model toggle** — Switch between NSL-KDD and CICIDS2017 from any page; all data refetches automatically
- **Welcome Modal** — Pulls real val/test accuracy and feature count from `/model-info`
- **Stat Cards** — Real DB counts in API mode; count-up animation on load
- **Traffic Chart** — Dual Y-axis area chart, 4 time ranges (6h / 12h / 24h / 7d), SQL-bucketed queries
- **Donut Chart** — Real category distribution from DB
- **Live Simulation** — Start/stop a background thread that samples the test set and writes predictions to DB at configurable intervals; all pages update in real time
- **Alerts Table** — Real API data, sortable columns, search, type filter pills, pagination, CSV export, auto-refresh
- **Predict Page** — Full KDD form (13 basic + 27 advanced) **or** full CICIDS form (20 basic + 49 advanced) depending on active model; quick-fill sample buttons; confidence bar; top-5 feature importance bars; session prediction history (last 8, re-usable)
- **Batch Page** — Drag-and-drop CSV upload; client-side column count validation before upload; category breakdown pills; downloadable results CSV
- **Globe** — 3D interactive globe (globe.gl) with pulsing rings, attack arcs, click-to-inspect pins; real top sources from DB
- **Threat Feed** — Seeds from DB on open; polls for new rows at the configured refresh interval; demo mode uses a randomised mock generator
- **Reports Page** — Weekly bar chart, monthly trend line, category breakdown bars, top source IPs table; PDF export via jsPDF
- **Model Info Page** — Real confusion matrix heatmap with heat-intensity coloring; horizontal feature importance bar chart; hyperparameters panel; val/test accuracy; F1 score
- **Command Palette** — `Ctrl K` spotlight search across all pages and attack type descriptions (model-aware: shows KDD or CICIDS attack types)
- **Settings** — Dark/light theme, auto-refresh toggle + interval (10s/30s/1min/5min), active model selector with loaded/not-loaded badges, API endpoint with connection test, mock/API toggle
- **Error Boundary** — Graceful crash recovery on all pages and globe
- **Skeleton Loaders** — Shimmer placeholders during data fetching
