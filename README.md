# NIDS — Network Intrusion Detection System

A machine-learning powered security dashboard that detects and classifies network intrusions in real time — built on the NSL-KDD dataset using XGBoost with SMOTE-balanced training, a FastAPI backend, PostgreSQL storage, and a React 19 frontend.

## Attack Categories

| Category | Color | Description |
|---|---|---|
| **Normal** | 🟢 `#22c55e` | Legitimate network traffic |
| **DoS** | 🔴 `#ef4444` | Denial of Service (neptune, smurf, back, teardrop…) |
| **Probe** | 🔵 `#38bdf8` | Surveillance / scanning (portsweep, nmap, ipsweep, satan…) |
| **R2L** | 🟡 `#f59e0b` | Remote-to-Local attacks (guess_passwd, ftp_write, imap…) |
| **U2R** | 🟣 `#a78bfa` | User-to-Root attacks (buffer_overflow, rootkit, perl…) |

> All 5 colors are defined once in `frontend/src/lib/colors.js` — imported everywhere, never hardcoded.

## Setup Instructions

### 1 — Clone & Install Python Dependencies

```bash
git clone https://github.com/Bhagyesh312/NIDS.git
cd NIDS

python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # Mac/Linux

pip install -r requirements.txt
```

### 2 — Configure the Database

Create a PostgreSQL database named `NIDS` in pgAdmin4, then:

```bash
cd backend
copy .env.example .env      # Windows
# cp .env.example .env      # Mac/Linux
```

Edit `.env` and set your PostgreSQL password:
```
DATABASE_URL=postgresql://postgres:YOUR_PASSWORD@localhost:5432/NIDS
```

### 3 — Run the ML Notebooks (in order)

```bash
# From the NIDS root directory:
jupyter lab

# Run these notebooks in order:
# 1. notebooks/01_eda_nslkdd.ipynb       (optional — EDA only)
# 2. notebooks/02_preprocessing.ipynb    (required — generates data/processed/)
# 3. notebooks/03_model_training.ipynb   (required — generates models/)
```

After notebook 03 completes you will have:
- `models/model.pkl` — the trained classifier
- `models/model_info.json` — real metrics served to the Model Info page

### 4 — Start the Backend

```bash
cd backend
uvicorn main:app --reload
# API docs: http://localhost:8000/docs
```

### 5 — Start the Frontend

```bash
cd frontend
npm install
npm run dev
# Dashboard: http://localhost:5173
```

## Frontend Features

- **Welcome Modal** — Intro screen; animations gate until "Enter Dashboard" is clicked
- **Flowing Menu** — Hamburger overlay with cascading wave animation
- **Stat Cards** — UIverse.io 3D ticket-style cards with scroll grid and 3D tilt
- **Count-up Numbers** — `requestAnimationFrame` + `easeOutExpo` easing
- **Traffic Chart** — Dual Y-axis area chart, time range filter (6h / 12h / 24h / 7d)
- **Donut Chart** — Animated spin-in with per-category progress bars
- **Alerts Table** — Real API data, sortable columns, search, pagination, CSV export, 30s auto-refresh
- **Predict Page** — Quick-fill sample buttons (DoS / Probe / Normal), animated confidence bar, top-5 feature bars
- **Batch Page** — Drag-and-drop CSV upload, category summary pills, downloadable results
- **Model Info Page** — Real confusion matrix heatmap + feature importance chart from trained model
- **Reports Charts** — Stacked bar chart with glow hover, monthly line chart
- **Globe** — 3D interactive globe with pulsing rings, arcs, glassmorphism popup
- **Threat Feed** — Terminal-style live event stream (slide-in panel)
- **Command Palette** — `Ctrl K` spotlight search across pages and attack types
- **Dark / Light Mode** — Settings toggle, CSS variables, saved to `localStorage`
- **Mock Mode** — Amber banner when backend is offline; 3D flip toggle (Demo ↔ API); live dot turns amber in demo mode
- **Error Boundary** — Graceful crash recovery on all pages
- **Skeleton Loaders** — Shimmer placeholders during data fetching
