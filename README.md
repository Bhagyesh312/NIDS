# NIDS — Network Intrusion Detection System

A machine-learning powered security dashboard that detects and classifies network intrusions in real time — built on the NSL-KDD dataset using XGBoost with SMOTE-balanced training, a FastAPI backend, PostgreSQL storage, and a React 19 frontend.

## Attack Categories

| Category | Color | Description |
|---|---|---|
| **Normal** | 🟢 `#22c55e` | Legitimate network traffic |
| **DoS** | 🔴 `#ef4444` | Denial of Service (neptune, smurf, back, teardrop…) |
| **Probe** | 🔵 `#38bdf8` | Surveillance / scanning (portsweep, nmap, ipsweep, satan…) |
| **R2L** | 🟡 `#f59e0b` | Remote-to-Local attacks (guess_passwd, ftp_write, imap…) |
| **U2R** | 🟣 `#a78bfa` | User-to-Root attacks (buffer_overflow, rootkit, perl…) |

> All 5 colors are defined once in `frontend/src/lib/colors.js` — imported everywhere, never hardcoded.

## Setup Instructions

### 1 — Clone & Install Python Dependencies

```bash
git clone https://github.com/Bhagyesh312/NIDS.git
cd NIDS

python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # Mac/Linux

pip install -r requirements.txt
```

### 2 — Configure the Database

Create a PostgreSQL database named `NIDS` in pgAdmin4, then:

```bash
cd backend
copy .env.example .env      # Windows
# cp .env.example .env      # Mac/Linux
```

Edit `.env` and set your PostgreSQL password:
```
DATABASE_URL=postgresql://postgres:YOUR_PASSWORD@localhost:5432/NIDS
```

### 3 — Run the ML Notebooks (in order)

```bash
# From the NIDS root directory:
jupyter lab

# Run in order:
# 1. notebooks/01_eda_nslkdd.ipynb       (optional — EDA only)
# 2. notebooks/02_preprocessing.ipynb    (required — generates data/processed/)
# 3. notebooks/03_model_training.ipynb   (required — generates models/)
```

After notebook 03 completes:
- `models/model.pkl` — trained XGBoost classifier
- `models/model_info.json` — real metrics served by the API

### 4 — Start the Backend

```bash
cd backend
uvicorn main:app --reload
# API docs: http://localhost:8000/docs
```

### 5 — Start the Frontend

```bash
cd frontend
npm install
npm run dev
# Dashboard: http://localhost:5173
```

## Frontend Features

- **Welcome Modal** — Pulls real val accuracy, test accuracy, and feature count from `/model-info`
- **Flowing Menu** — Hamburger overlay with cascading wave animation
- **Stat Cards** — Real DB counts in API mode; count-up animation with `easeOutExpo`
- **Traffic Chart** — Dual Y-axis area chart, time range filter (6h / 12h / 24h / 7d), real DB data
- **Donut Chart** — Real category distribution from DB, animated spin-in
- **Alerts Table** — Real API data, sortable columns, search, pagination, CSV export, configurable auto-refresh
- **Predict Page** — All 40 KDD features (13 basic + 27 advanced in expandable groups), sample fill buttons, animated confidence bar, top-5 feature bars, session prediction history (last 8, re-usable)
- **Batch Page** — Drag-and-drop CSV upload, category summary pills, downloadable results
- **Model Info Page** — Real confusion matrix heatmap + feature importance chart from trained model; shows val/test accuracy and F1
- **Reports Page** — All values real in API mode; monthly trend from DB, detection rate from `model_info.json`
- **Globe** — 3D interactive globe with pulsing rings, arcs; real top sources panel from DB
- **Threat Feed** — In API mode: seeds from DB predictions and polls for new rows; in demo mode: random mock generator
- **Command Palette** — `Ctrl K` spotlight search across pages and attack types
- **Dark / Light Mode** — Full CSS variable system; switching applies to all surfaces and text
- **Auto-refresh Settings** — Toggle on/off + interval (10s / 30s / 1min / 5min); persisted to `localStorage`; controls Dashboard, Alerts, ThreatFeed simultaneously
- **Mock Mode** — Demo ↔ API toggle; amber banner when in demo mode; all pages fall back gracefully when backend is offline
- **Error Boundary** — Graceful crash recovery on all pages
- **Skeleton Loaders** — Shimmer placeholders during data fetching
